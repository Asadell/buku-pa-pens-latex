\chapter*{ABSTRAK}
\addcontentsline{toc}{chapter}{ABSTRAK}
\thispagestyle{fancy}

Aplikasi FoodLAB yang digunakan untuk pemesanan makanan daring di Politeknik Elektronika Negeri Surabaya (PENS) menghadapi tantangan signifikan dalam hal observability dan pemulihan insiden pada sisi backend-nya. Observability merupakan kemampuan suatu sistem untuk memperlihatkan kondisi internalnya melalui data telemetri berupa metrics, log, dan traces, sehingga operator dapat memahami perilaku sistem secara menyeluruh. Sistem monitoring konvensional yang berbasis threshold statis dan respons manual terbukti tidak memadai untuk mendeteksi anomali secara akurat, sehingga menghasilkan false-positive rate yang tinggi serta memperpanjang Mean Time to Detect (MTTD) dan Mean Time to Recovery (MTTR). Untuk mengatasi permasalahan tersebut, penelitian ini merancang dan mengimplementasikan framework Agentic DevOps Monitoring Agents (ADMA). ADMA adalah sistem multi-agent otonom berbasis kecerdasan buatan yang mampu melakukan observability, root cause analysis, dan self-healing pada infrastruktur backend FoodLAB. Sistem ADMA terdiri dari dua komponen utama. Pertama, sebuah Go binary yang berjalan di server target dan bertugas melakukan monitoring secara real-time mencakup PostgreSQL, Docker container logs, dan resource utilization sistem. Komponen ini juga melakukan diagnosis berbasis Large Language Models (LLM) serta eksekusi remediasi otomatis. Kedua, sebuah backend control plane berbasis C\# ASP.NET Core yang berfungsi sebagai pusat manajemen, persistensi data, dan antarmuka REST API. Kedua komponen berkomunikasi secara asinkron melalui NATS messaging. Hasil implementasi ADMA pada infrastruktur FoodLAB diharapkan dapat meningkatkan kemampuan deteksi anomali secara proaktif, mempercepat proses diagnosis dan pemulihan insiden, serta mengurangi ketergantungan pada intervensi manual dalam pengelolaan sistem.
